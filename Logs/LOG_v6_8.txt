BUG FIX: COMPRESS-PDF (v6.8)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
CAUSA: pypdf._xobj_image_helpers importa PIL (Pillow) en el módulo.
       El código anterior instalaba solo 'pypdf'. Al intentar
       img.replace(img.image, quality=N), PIL no estaba disponible.

FIX EN initPyodide():
  ANTES: await micropip.install('pypdf')
  DESPUÉS: await micropip.install(['pypdf','pillow'])

FIX EN el código Python:
  for page in writer.pages:
      for img in page.images:
          try:
              img.replace(img.image, quality=int(img_quality))
          except Exception:
              pass  # skip images that can't be recompressed

El try/except es importante porque algunos PDFs tienen imágenes
en formatos que pypdf no puede re-comprimir (e.g. JBIG2, CCITT).
Sin try/except, un solo imagen problemática rompía toda la compresión.

ROTATE-PDF — ARQUITECTURA NUEVA (v6.8)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Layout: 2 columnas en desktop — izquierda controles, derecha preview
Preview: PDF.js renderiza la página actual con rotación acumulada aplicada
         Navegación prev/next para ver diferentes páginas

Rotación:
  pageRotations = {} — dict {pageNum: angle} — ángulo acumulado
  rotatePreview(delta): suma delta al ángulo de todas las páginas target
                        y re-renderiza el preview inmediatamente
  Botones: Left (-90°), Flip (180°), Right (+90°)
  Scope: 'all' (todas las páginas) o 'range' (páginas específicas)

Download (doRotate()):
  Pasa rot_map_json = {pageIndex_0based: angle} a Python
  Python itera reader.pages, aplica page.rotate(angle) y construye writer
  Solo se rotan páginas con angle != 0

Comportamiento UX:
  - Al subir PDF: preview se carga inmediatamente (no espera Pyodide)
  - Al hacer clic en botón: preview se actualiza al instante (solo re-render JS)
  - Pyodide carga en background mientras el usuario ve el preview
