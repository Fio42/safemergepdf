GUÍA COMPLETA: CLOUDFLARE SETUP PASO A PASO
════════════════════════════════════════════════════════════

── PASO 1: Crear cuenta Cloudflare
   → https://dash.cloudflare.com/sign-up
   → Email + contraseña → Verificar email

── PASO 2: Añadir dominio noservertools.com
   → Dashboard → "Add a Site" → escribir noservertools.com
   → Elegir plan FREE → Continue
   → Cloudflare escanea tus DNS automáticamente

── PASO 3: Importar DNS de Namecheap → Cloudflare
   ANTES de cambiar nameservers, verifica que CF importó estos registros:

   TIPO   NOMBRE   VALOR                          PROXY

   A      @        185.199.111.153                ✅ Proxied (naranja)
   CNAME  www      USERNAME.github.io             ✅ Proxied (naranja)

   (Estos son los IPs de GitHub Pages — si no aparecen, añádelos manualmente)

   IMPORTANTE: el registro CNAME www debe apuntar a TU usuario:
   fiocc.github.io  (reemplaza con tu usuario real de GitHub)

── PASO 4: Cambiar nameservers en Namecheap
   → Namecheap → Dashboard → noservertools.com → Manage
   → "Nameservers" → seleccionar "Custom DNS"
   → Pegar los 2 nameservers que Cloudflare te dio, ejemplo:
       aria.ns.cloudflare.com
       bob.ns.cloudflare.com
   → Save ✅
   → Propagación: 5 min – 48 horas (normalmente <30 min)

── PASO 5: Configuración SSL en Cloudflare
   → SSL/TLS → Overview → modo "Full" (no Full Strict, GitHub Pages no tiene cert propio)
   → Edge Certificates → Always Use HTTPS → ON
   → Minimum TLS Version → TLS 1.2

── PASO 6: Activar Web Analytics
   → Cloudflare Dashboard → tu dominio → "Web Analytics" (menú izquierdo)
   → "Add a site" → noservertools.com
   → Cloudflare genera un TOKEN único (aspecto: abc123def456...)
   → Copia ese token

── PASO 7: Insertar token en todos los HTML
   Busca en cada archivo:
     REPLACE_WITH_YOUR_CF_TOKEN
   Reemplázalo con tu token real. Ejemplo final:
