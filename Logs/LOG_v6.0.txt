RECETAS TECNICAS — HERRAMIENTAS ACTIVAS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Todas usan Pyodide v0.25.0 + pypdf, excepto pdf-to-jpg.

MERGE PDF:
  Pyodide + pypdf
  PdfWriter().append(BytesIO(bytes)) por cada archivo
  writer.write(output_buffer)
  Descarga como .pdf

COMPRESS PDF:
  Pyodide + pypdf
  compress_identical_objects=True
  Por cada imagen: img.replace(quality=N) con N segun nivel elegido
  Descarga como .pdf comprimido

SPLIT PDF:
  Pyodide + pypdf
  PdfWriter().add_page(page) por cada pagina del rango elegido
  writer.write(output) -> descarga como .pdf

PDF -> JPG:
  PDF.js (NO Pyodide)
  page.render() -> canvas -> canvas.toDataURL('image/jpeg')
  Descarga cada pagina como .jpg

JPG -> PDF:
  Pyodide + Pillow
  Image.open(BytesIO(bytes)) -> img.save(buf, format='PDF')
  PdfWriter().append(BytesIO(pdf_bytes))
  Descarga como .pdf

ROTATE PDF (PROXIMA):
  Pyodide + pypdf
  page.rotate(N) donde N es 90, 180, 270
  PdfWriter con todas las paginas rotadas
  Descarga como .pdf

PROTECT PDF (FUTURA):
  Pyodide + pypdf
  writer.encrypt(user_password, owner_password, use_128bit=True)
  Descarga como .pdf protegido

UNLOCK PDF (FUTURA):
  Pyodide + pypdf
  reader.decrypt(password)
  writer.clone_reader_document_root(reader)
  Descarga como .pdf sin contrasena

VIEW PDF METADATA (FUTURA — diferenciador privacidad):
  Pyodide + pypdf
  reader.metadata -> dict con Author, Creator, Producer, Subject, etc.
  Mostrar en tabla visible, sin descargar nada

REMOVE PDF METADATA (FUTURA — diferenciador privacidad):
  Pyodide + pypdf
  writer.add_metadata({}) -> limpia todos los campos
  Descarga como .pdf sin metadatos

REDACT PDF (FUTURA — diferenciador privacidad):
  PDF.js: renderizar paginas
  Canvas API: usuario dibuja rectangulos sobre texto sensible
  pypdf o jsPDF: aplicar rectangulos negros permanentes
  NOTA: requiere diseno de UI cuidadoso (drag-to-select)

SISTEMA DE I18N (MULTIIDIOMA DE INTERFAZ)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Aplica SOLO a textos de la interfaz. NO a contenido de documentos del usuario.

Idiomas soportados (objetivo):
  en (English) — fallback por defecto
  es (Espanol)
  fr (Francais)
  de (Deutsch)
  pt (Portugues)
  it (Italiano)
  ja (Japanese)
  ko (Korean)
  zh (Chinese Simplified)

Implementacion recomendada:
  - Objeto I18N con clave: { en: '...', es: '...', fr: '...', ... }
  - Deteccion: navigator.language.slice(0,2).toLowerCase()
  - Todos los textos del HTML: atributo data-i18n="clave"
  - Funcion applyI18n() recorre todos los elementos y aplica traducciones
  - Fallback automatico a 'en' si el idioma del navegador no esta en I18N

Patrón de implementacion:
  const I18N = {
    title: { en: 'Merge PDF', es: 'Unir PDF', fr: 'Fusionner PDF', ... },
    ...
  };
  function applyI18n(lang) {
    const L = I18N;
    document.querySelectorAll('[data-i18n]').forEach(el => {
      const key = el.dataset.i18n;
      el.textContent = (L[key] && L[key][lang]) || (L[key] && L[key]['en']) || key;
    });
  }
  const uiLang = navigator.language.slice(0,2).toLowerCase();
  applyI18n(['en','es','fr','de','pt','it','ja','ko','zh'].includes(uiLang) ? uiLang : 'en');

CAMBIO 1
  Accion:      Eliminacion de translate-pdf
  Archivo:     translate-pdf/index.html (eliminar carpeta completa)
               index.html (quitar card + link)
               sitemap.xml (quitar entrada)
               nav/footer de todas las paginas (quitar link)
  Descripcion: La herramienta translate-pdf enviaba texto del usuario a la
               API de MyMemory (servicio externo). Esto contradice el
               posicionamiento de privacidad total del sitio. Aunque el
               archivo no salia del dispositivo, los datos de texto si lo
               hacian, lo que debilita el mensaje "Your files never leave
               your computer." Ademas, la herramienta usaba Pyodide
               innecesariamente (carga ~10MB) y tenia dependencias de
               MyMemory, LibreTranslate y jsPDF que aumentaban el peso
               y la complejidad.
  Motivo:      Alineacion con el posicionamiento de privacidad total.
               Reduccion de complejidad y bundle size. Eliminacion de
               dependencia en servicios externos.
  Impacto SEO: Neutro. translate-pdf generaba poco trafico organico
               comparado con herramientas de mayor volumen de busqueda.
  Impacto UX:  Positivo. El sitio queda mas coherente con su propuesta
               de valor. El usuario no ve contradicciones.
  Impacto perf: Positivo. Se eliminan 3 CDN scripts y ~10MB de Pyodide
               en la herramienta mas pesada del sitio.

CAMBIO 2

  Archivo:     index.html, todos los index.html de herramientas
  Descripcion: Cambio de tagline de "Tools that run inside your browser.

               computer." como mensaje H1 o sub-headline del hero.

  Motivo:      Mensaje mas directo, emocional y diferenciador.

  Impacto SEO: Positivo. Mejor para keywords de intencion de privacidad.
