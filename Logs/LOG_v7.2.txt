ERROR RAÍZ REAL — Identificado en consola del navegador
════════════════════════════════════════════════════════════

ERROR EXACTO EN CONSOLA:
  CompileError: WebAssembly.instantiateStreaming(): Compiling or
  instantiating WebAssembly module violates the following Content
  Security Policy directive because 'unsafe-eval' is not an allowed
  source of script in the following Content Security Policy directive.

CAUSA:
  Pyodide necesita OBLIGATORIAMENTE estas dos directivas en script-src:
    'unsafe-eval'        — para ejecutar código JS generado por WASM
    'wasm-unsafe-eval'   — para compilar módulos WebAssembly (Chrome 97+)

  El <meta CSP> que teníamos NO las incluía.
  El navegador bloqueaba WebAssembly.instantiateStreaming() silenciosamente.
  loadPyodide() quedaba colgado infinitamente → "Loading engine" forever.

  TAMBIÉN: Los meta tags <X-Frame-Options> y <X-Content-Type-Options>
  generaban errores en consola porque esas directivas SOLO funcionan
  como HTTP headers del servidor, NO como meta tags HTML.

SOLUCIÓN DEFINITIVA:
  1. Eliminar TODOS los <meta http-equiv="Content-Security-Policy"> de
     todos los archivos HTML. En GitHub Pages (sitio estático) el CSP
     via meta tag:
       a) No puede incluir 'wasm-unsafe-eval' sin romper todo
       b) No da protección real contra ataques externos
       c) Rompe Pyodide, PDF.js workers y WebAssembly

  2. Eliminar todos los meta tags de X-Frame-Options y X-Content-Type-Options
     (solo funcionan como headers HTTP del servidor)

  3. Las cabeceras de seguridad correctas van en el archivo _headers
     (Netlify) o vía GitHub Actions para GitHub Pages.
