FLUJO COMPLETO DE LA HERRAMIENTA:

  PASO 1 — UPLOAD
    - Usuario arrastra o selecciona archivo PDF o TXT
    - Validacion: tipo de archivo + maximo 20 MB
    - Se guarda en currentFile
    - Se muestra chip con nombre y peso del archivo
    - Se avanza a pantalla de seleccion de idiomas

  PASO 2 — SELECCION DE IDIOMAS
    - Dos selectores: FROM (idioma origen) y TO (idioma destino)
    - 27 idiomas disponibles (ver lista abajo)
    - Auto-detect ELIMINADO — MyMemory no lo soporta, causaba error INVALID LANGUAGE PAIR
    - Defaults inteligentes segun idioma de la interfaz:
        UI en espanol -> FROM: Espanol, TO: Ingles
        UI en ingles  -> FROM: Espanol, TO: Ingles
    - Boton swap para invertir el par
    - Validacion: no permitir mismo idioma en FROM y TO
    - Aviso amarillo: "Solo el texto (no el archivo) se envia a la API de traduccion"

  PASO 3 — EXTRACCION (100% local, nada sale del dispositivo)
    Si archivo es TXT:
      - file.text() directo
      - isPdfFile = false

    Si archivo es PDF (funcion extractWithPdfJs):
      - Configura PDF.js worker
      - Carga el PDF con pdfjsLib.getDocument({ data: arrayBuffer })
      - Por cada pagina (escala SCALE = 2.0 para alta calidad):
          a. page.getViewport({ scale: 2.0 })
          b. Crea canvas, renderiza pagina completa con page.render()
          c. canvas.toDataURL('image/jpeg', 0.92) -> imageDataUrl
          d. page.getTextContent() -> extrae items de texto con posiciones
          e. Normaliza coordenadas: x = tx/vpWidth, y = 1-(ty/vpHeight)
             (PDF tiene origen abajo-izquierda, se flipea el eje Y)
          f. Normaliza fontSize = |scaleY| / vpHeight
          g. Guarda en pdfPageData[]
      - Concatena todo el texto para traduccion
      - isPdfFile = true

  PASO 4 — TRADUCCION (API externa — solo texto, no archivo)
    - API: MyMemory (https://api.mymemory.translated.net/get)
    - Sin API key, sin cuenta, completamente gratuita
    - Limite free: ~5000 chars/request, ~50 requests/dia por IP
    - El texto se divide en chunks de <= 450 caracteres (por oracion)
    - Entre cada chunk: 300ms de delay para no exceder rate limit
    - Formato del par de idiomas: "es|en", "fr|de", etc. (NUNCA con autodetect)
    - Los chunks traducidos se van mostrando en tiempo real en el panel

  PASO 5 — DESCARGA PDF

    MODO VISUAL (isPdfFile = true, pdfPageData disponible):
      Funcion buildVisualPdf():
      - Crea jsPDF con las dimensiones exactas de cada pagina del PDF original
      - Por cada pagina:
          1. doc.addImage() — pone la imagen JPEG de la pagina como fondo completo
             Esto preserva: fondos de color, logos, imagenes, iconos, layout
          2. Mapeo proporcional de texto traducido:
             - Cuenta total de text items en todas las paginas
             - Para cada text item de la pagina original, calcula que porcion
               del texto traducido le corresponde (por proporcion de indice)
             - transWords distribuidos proporcionalemente sobre origWords
          3. Por cada text item:
             - Convierte coordenadas normalizadas de vuelta a puntos PDF
             - Calcula fontSize en puntos (clampeado entre 6 y 72)
             - Dibuja rectangulo blanco semitransparente (opacity 0.75) detras
               del texto para que sea legible sobre cualquier fondo
             - Dibuja el texto traducido en esa posicion (opacity 1.0)
      - doc.save(base + '-translated.pdf')

    MODO TEXTO (isPdfFile = false, archivo TXT):
      Funcion buildTextPdf():
      - PDF A4 clasico con margenes
      - Header: titulo del archivo + par de idiomas
      - Cuerpo: parrafos con espaciado correcto
      - Footer en cada pagina: nombre archivo + numeracion N/Total
      - doc.save(base + '-translated.pdf')

    FALLBACK:
      Si jsPDF falla por cualquier razon -> downloadTxt('translated')
      El boton muestra "Saved as .txt (error)"

  DESCARGA TXT (siempre disponible):
    - Original: Blob text/plain + a.click() + URL.revokeObjectURL()
    - Traducido: Blob text/plain + a.click() + URL.revokeObjectURL()

LIMITACIONES CONOCIDAS (sin solucion posible sin servidor):
  - El texto traducido puede ser mas largo que el original y superponerse
  - Las fuentes del PDF original no se pueden replicar exactamente
  - Imagenes dentro del PDF que NO son de fondo (inline) se ven en el fondo
    pero el texto encima puede taparlas parcialmente
  - PDFs escaneados (imagen pura, sin texto) no tienen texto extraible
    -> Se muestra error claro: "This PDF appears to be image-based (scanned)"
  - Idiomas con caracteres especiales (chino, japones, arabe) pueden no
    renderizar correctamente en jsPDF sin embeber fuentes especiales
  - Rate limit de MyMemory: documentos muy largos pueden fallar (~50 req/dia)

Todas usan Pyodide v0.25.0 + pypdf, excepto pdf-to-jpg y translate-pdf.

MERGE PDF:
  Pyodide + pypdf
  PdfWriter().append(BytesIO(bytes)) por cada archivo
  writer.write(output_buffer)
  Descarga como .pdf

COMPRESS PDF:
  Pyodide + pypdf
  compress_identical_objects=True
  Por cada imagen: img.replace(quality=N) con N segun nivel elegido
  Descarga como .pdf comprimido

SPLIT PDF:
  Pyodide + pypdf
  PdfWriter().add_page(page) por cada pagina del rango elegido
  writer.write(output) -> descarga como .pdf

PDF -> JPG:
  PDF.js (NO Pyodide)
  page.render() -> canvas -> canvas.toDataURL('image/jpeg')
  Descarga cada pagina como .jpg

JPG -> PDF:
  Pyodide + Pillow
  Image.open(BytesIO(bytes)) -> img.save(buf, format='PDF')
  PdfWriter().append(BytesIO(pdf_bytes))
  Descarga como .pdf

TRANSLATE PDF:
  PDF.js: render paginas como JPEG + extraccion texto con posiciones
  MyMemory API: traduccion del texto en chunks de 450 chars
  jsPDF: genera PDF con imagen de pagina + texto traducido superpuesto
  Ver seccion completa arriba.
