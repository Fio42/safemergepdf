TRANSLATE-PDF — ARQUITECTURA TECNICA v5.8
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
EXTRACCION (100% local):
  - PDF: Pyodide + pypdf -> page.extract_text() -> string
  - TXT: file.text() directo, sin Pyodide
  - Limite: 20 MB por archivo

TRADUCCION (API externa - solo texto):
  - API: MyMemory (api.mymemory.translated.net/get)
  - Sin API key, sin cuenta, gratuita
  - Limite free: ~5000 chars/request, ~50 requests/dia
  - Chunking: texto dividido en partes <= 450 chars por oracion
  - Rate limiting: 300ms delay entre chunks
  - Lang pair format: "en|es" — siempre con codigo explicito, sin autodetect

IDIOMAS SOPORTADOS (28 total):
  en, es, fr, de, pt, it, nl, pl, ru, tr,
  ar, zh, ja, ko, sv, da, fi, no, ro, uk, cs, hu, el, he,
  id, th, vi (+ autodetect eliminado en v5.7 — causaba INVALID LANGUAGE PAIR)

UI MULTIIDIOMA (5 idiomas de interfaz):
  English, Espanol, Francais, Deutsch, Portugues
  Deteccion: navigator.language (sin API, 100% local)
  Strings: objeto I18N con todas las traducciones
  Todos los textos tienen data-i18n attribute

DESCARGA v5.8:
  - TXT original:    Blob text/plain + URL.createObjectURL
  - TXT traducido:   Blob text/plain + URL.createObjectURL
  - PDF traducido:   jsPDF v2.5.1 via CDN (cdnjs.cloudflare.com) — PDF REAL
    * Genera .pdf directo, sin pasar por HTML intermedio
    * Titulo del documento en header
    * Par de idiomas (FROM -> TO) + "Translated by NoServerTools.com"
    * Linea separadora debajo del header
    * Parrafos con espaciado correcto (10pt entre parrafos)
    * Saltos de pagina automaticos cuando el contenido no cabe
    * Footer en cada pagina: nombre archivo a la izquierda, N/Total a la derecha
    * Linea separadora encima del footer
    * Fallback a .txt si jsPDF falla por cualquier razon

  LIMITACION CONOCIDA (sin solucion posible sin servidor):
    - Imagenes del PDF original no se incluyen (pypdf extrae solo texto)
    - Layout de columnas exacto no se puede reconstruir (texto es lineal)
    - PDFs escaneados (imagen) no tienen texto extraible -> error claro en UI

DEPENDENCIAS NUEVAS EN v5.8:
  <script src="https://cdnjs.cloudflare.com/ajax/libs/jspdf/2.5.1/jspdf.umd.min.js"></script>
  Agregar ANTES del script de Pyodide en el HTML.
  Acceso via: const { jsPDF } = window.jspdf;
