TRANSLATE-PDF — ARQUITECTURA TECNICA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
EXTRACCION (100% local):
  - PDF: Pyodide + pypdf -> page.extract_text() -> string
  - TXT: file.text() directo, sin Pyodide
  - Limite: 20 MB por archivo

TRADUCCION (API externa - solo texto):
  - API: MyMemory (api.mymemory.translated.net/get)
  - Sin API key, sin cuenta, gratuita
  - Limite free: ~5000 chars/request, ~50 requests/dia
  - Chunking: texto dividido en partes <= 450 chars por oracion
  - Rate limiting: 300ms delay entre chunks
  - Lang pair format: "en|es" o "|es" para autodetect

IDIOMAS SOPORTADOS (28 total):
  autodetect, en, es, fr, de, pt, it, nl, pl, ru, tr,
  ar, zh, ja, ko, sv, da, fi, no, ro, uk, cs, hu, el, he,
  id, th, vi

UI MULTIIDIOMA (5 idiomas de interfaz):
  English, Espanol, Francais, Deutsch, Portugues
  Deteccion: navigator.language (sin API, 100% local)
  Strings: objeto I18N con todas las traducciones
  Todos los textos tienen data-i18n attribute

DESCARGA:
  - TXT original: Blob text/plain + URL.createObjectURL
  - TXT traducido: Blob text/plain + URL.createObjectURL
  - "PDF": genera HTML con el texto y se descarga como .html
    (usuario puede abrir y File > Print > Save as PDF)
  NOTA FUTURA: para PDF real usar jsPDF o reportlab via Pyodide

LIMITACION CONOCIDA:
  - PDFs basados en imagenes (escaneados) no tienen texto extraible
  - Mostrar error claro: "This PDF appears to be image-based (scanned)"
  - Solucion futura: OCR con tesseract.js
